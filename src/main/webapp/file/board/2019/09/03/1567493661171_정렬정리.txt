1. 대부분의 정렬 알고리즘들은 입력을 위한 메모리 공간과 O(1) 크기의 메모리 공간만을 사용하면서 정렬수행
-O(1) 크기의 메모리 공간이란 입력 크기 n과 상관없는 크기의 공간(예를 들어, 변수, 인덱스 등)을 의미

2. 합병 정렬의 공간 복잡도: O(n)
- 입력을 위한 메모리 공간 (입력 배열)외에 추가로 입력과 같은 크기의 공간 (임시 배열)이 별도로 필요.
- 2개의 정렬된 부분을 하나로 합병하기 위해, 합병된 결과를 저장할 곳이 필요하기 때문

응용
1. 합병 정렬은 외부 정렬의 기본이 되는 정렬 알고리즘
2. 연결 리스트에 있는 데이터를 정렬할 때에도 퀵정렬이나 힙 정렬보다 훨씬 효율적
3. 멀티코어 CPU와 다수의 프로세서로 구성된 그래픽 처리 장치의 등장으로
정렬 알고리즘을 병렬화하는 데에 합병 정렬 알고리즘이 활용

성능향상 방법
입력의 크기가 매우 클때, 퀵 정렬의 성능을 더 향상 시키기 위해서, 
삽입 정렬을 동시에 사용
- 입력의 크기가 작을 때에는 퀵 정렬이 삽입 정렬보다 빠르지만은 않다.
ㅇ 퀵정렬은 재귀 호출로 수행되기때문
- 부분문제의 크기가 작아지면 예를 들어, 25에서 50이되면), 더 이상의 분할(재귀 호출)을 중단하고
삽입 정렬을 사용한다.

응용
-퀵 정렬은 커다란 크기의 입력에 대해서 가장 좋은 성능을 보이는 정렬 알고리즘이다.
-퀵 정렬은 실질적으로 어느 정렬 알고리즘보다 좋은 성능을 보인다.
-생물 정보 공학에서 특정
  유전자를 효율적으로 찾는데 접미 배열과 함께 퀵 정렬이 활용된다.
